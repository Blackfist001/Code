Services web V3 (SOAP)


Ci-dessous, vous trouverez toutes les informations des différentes méthodes que vous pouvez utiliser via nos services web.
Nous ne supportons que les requêtes qui utilisent la version TLS 1.2 ou supérieure.
Liens URL

    https://centreleonarddefrance.smartschool.be/Webservices/V3
    https://centreleonarddefrance.smartschool.be/Webservices/V3?wsdl

Exemples d'applications en PHP et en C#

Nous vous offrons quelques exemples concrets de code de programmation et C# et en PHP qui utilisent les services web et OAuth:

    Application C# qui permet de modifier le statut et le mot de passe d'un utilisateur.
    Application PHP qui permet de modifier le mot de passe d'un utilisateur.
    Application PHP pour commander des sandwich, authentication avec OAuth.

Le code source de toutes les applications peuvent être téléchargées comme source d'inspiration. Il ne sont pas prêts à l'emploi, mais peuvent être utilisés comme premier pas ou comme exemple.

Détails des méthodes
addCourse

Cette méthode permet de créer des cours dans Smartschool. Le fichier xml exige 3 paramètres:

    string $accesscode: mot de passe services web
    string $coursename: nom du cours
    string $coursedesc: description du cours
    int $visibility: Visibilité (facultatif: '1' = visible (valeur par défaut) ou '0' = invisible)

addCourseStudents

Cette méthode permet d'ajouter des classes à un cours. Le fichier xml exige 4 paramètres:

    string $accesscode: mot de passe services web
    string $coursename: nom du cours
    string $coursedesc: description du cours
    string $groupIds: liste CSV avec un (des) code(s) classe ou un (des) code(s) groupe unique(s)

addCourseTeacher

Cette méthode permet d'ajouter des enseignants à un cours. Le fichier xml exige 5 paramètres:

    string $accesscode: mot de passe services web
    string $coursename: nom du cours
    string $coursedesc: description du cours
    string $internnummer: champ utilisateur unique
    string $userlist: liste CSV avec un champ utilisateur unique

addHelpdeskTicket

Cette méthode permet de créer un ticket dans le module Helpdesk. Utilisez la méthode 'getHelpdeskMiniDbItems' pour récupérer tous les éléments des minibases de données qui peuvent être utilisé dans le module Helpdesk.

    string $accescode: mot de passe services web
    string $title: titre du ticket Helpdesk
    string $description: description du ticket Helpdesk
    integer $priority: priorité du ticket Helpdesk. Valeur entre 1 et 5 (valeur par défaut est 3).
    string $miniDbItem: élément de la minibase de données
    string $userIdentifier: champ utilisateur unique

changeGroupOwners

Cette méthode permet de modifier le(s) titulaire(s) de classe.

    string $accesscode: mot de passe services web
    string $code: code classe ou code groupe unique
    string $userlist: liste CSV avec un champ utilisateur unique

changeInternNumber

Cette méthode permet de modifier le numéro interne d'un utilisateur en utilisant l'identifiant comme champ clé.

    string $accesscode: mot de passe services web
    string $username: identifiant
    string $newInternNumber: nouveau numéro interne

changePasswordAtNextLogin

Cette méthode permet d'obliger un compte principal ou secondaire de modifier son mot de passe lors de la prochaine connexion.

    string $accescode: mot de passe services web
    string $userIdentifier: champ utilisateur unique
    integer $accountType: type du compte ('0' = hoofdaccount, '1' = co-account 1, '2' = co-account 2…)

changeUsername

Cette méthode permet de modifier l'identifiant d'un utilisateur en utilisant le numéro interne comme champ clé.

    string $accesscode: mot de passe services web
    string $internNumber: numéro interne
    string $newUsername: nouvel identifiant

checkStatus

Cette méthode permet de demander le statut de la demande de synchronisation de la méthode startSkoreSync.

    string $accesscode: Mot de passe services web
    string $serviceID: Le code unique de la demande de synchronisation de la méthode startSkoreSync

Cette méthode donne en retour le statut du travail de synchronisation:

    no_status: il n'y a pas de travail trouvé pour ce serviceID
    busy: le travail est en cours
    done: le travail est terminé
    error: le travail n'est pas terminé à cause d'une erreur lors du travail

clearGroup

    string $accesscode: mot de passe services web
    string $group: code classe ou code groupe unique
    string $officialDate: date officielle (facultatif: sans date officielle, l'éventuel changement de classe devra encore être confirmé dans Smartschool; format: YYYY-MM-DD)

delClass

Cette méthode permet de supprimer des groupes, classes et des classes officielles de Smartschool. Cette méthode peut être appelée avec 1 paramètre supplémentaire: $code. Il suffit d'identifier le groupe. Les membres de ce groupe ne sont pas supprimés de Smartschool.

    string $accesscode: mot de passe services web
    string $code: code classe ou code groupe unique

delUser

Cette méthode permet de supprimer l'utilisateur indiqué de Smartschool. Les données suivantes sont exigées:

    string $accesscode: mot de passe services web
    string $userIdentifier: champ utilisateur unique
    string $officialDate: date officielle (facultatif: sans date officielle, le changement de classe doit être confirmé dans Smartschool; format: AAAA-MM-JJ)

forcePasswordReset

Cette méthode permet d'obliger un certain utlisateur de modifier son mot de passe à la prochaine connexion.

    integer $accountType: type de compte ('0' = compte principal, '1' = compte secondaire 1 ou '2' = compte secondaire 2 …)

saveClass

Cette méthode permet de créer ou de modifier des classes ou des classes officielles dans Smartschool. Il n'est pas possible de modifier le nom des classes existantes (officielles) ou de les déplacer sous un autre groupe ou classe principale. Le fichier XML exige 6 paramètres et 3 paramètres facultatifs.

Si aucune classe (officielle) n'apparaît avec un string identification ($name) fourni, Smartschool va créer une nouvelle classe ( officielle ). S'il exsiste déjà une classe (officielle) avec un string identification fourni, les paramètres existants seront alors modifiés.

    string $accesscode: mot de passe services web
    string $name: nom de la classe ou du groupe
    string $desc: description
    string $code: code classe ou code groupe unique
    string $parent: code classe ou code groupe unique du groupe parent (vide ou '0' pour le parent actuel)
    Il n'est pas possible de déplacer une classe (officielle) existante via les API.
    string $untis: code horaire
    string $instituteNumber: numéro FASE (facultatif: seulement pour la création d'une classe officielle)
    string $adminNumber: numéro du groupe administratif (facultatif: seulement pour la création d'une classe officielle)
    string $schoolYearDate: ce string adapte le numéro FASE et le numéro du groupe administratif pour l'année scolaire attribuée (si l'année scolaire n'est pas attribuée ce string sera adapté pour l'année scolaire actuelle)

saveUser

Ajoute un utilisateur ou modifie un utilisateur existant.

Lorsque la méthode saveUser est appelée, le champ unique paramétré (userIdentifier) est alors contrôlé. Si aucun compte n'existe avec le champ unique paramétré, un nouveau compte sera alors ajouté. Si un compte avec un champ unique paramétré existe déjà, les données de ce compte seront alors mises à jour avec les données fournies.

    string $accesscode: mot de passe services web
    string $internnumber: numéro interne
    string $username: identifiant (au moins 2 caractères)
    string $passwd1: mot de passe du compte principal
    Il faut obligatoirement indiquer ce mot de passe pour les nouveaux utilisateurs. Pour les utilisateurs existants, il est possible de laisser ce string vide. Le mot de passe de cet utilisateur ne sera alors pas modifié.
    Retrouvez ici toutes les informations concernant les critères d'un mot ou d'une phrase de passe.
    string $passwd2: mot de passe compte secondaire (facultatif)
    string $passwd3: mot de passe deuxième compte secondaire (facultatif)
    string $name: prénom (au moins 1 caractère)
    string $surname: nom de famille (au moins 1 caractère)
    string $extranames: prénoms supplémentaires (facultatif)
    string $initials: initiales (facultatif)
    string $sex: sexe: 'm' ou 'f' (facultatif)
    string $birthday: date de naissance (facultatif; format AAAA-MM-JJ ou JJ-MM-AAAA)
    string $birthplace: lieu de naissance (facultatif)
    string $birthcountry: pays de naissance (facultatif)
    string $address: adresse (facultatif)
    Le numéro de la maison et le numéro de la boîte peuvent être indiqués dans le paramètre '$address'. Lorsque l'adresse a la forme <nom de la rue> espace <numéro de la maison> espace <numéro de la boîte>, le saveUser service la reconnaîtra et insérera les données dans les champs corrects de la base de données. Par exemple: Rue de l'église 22 ou Rue de l'église 22 boîte 1. Parfois on utilise une barre oblique pour séparer le numéro de la rue et le numéro de la boîte. Rue de l'église 22/1 devient Rue de l'église 22 boîte 1.
    string $postalcode: code postal (facultatif)
    string $location: ville/commune (facultatif)

saveUserToClass

Cette méthode permet de déplacer l'utilisateur dans une (nouvelle) classe officielle. Un élève ne peut être associé qu'à une seule classe officielle. La date officielle est facultative.

    string $accesscode: mot de passe services web
    string $userIdentifier: champ utilisateur unique
    string $class: code classe ou code groupe unique
    string $officialDate: date officielle (facultatif: sans date officielle, le changement de classe doit être confirmé dans Smartschool; format: AAAA-MM-JJ)

saveUserToClasses

Cette méthode permet d'associer un utilisateur à plusieurs classes/groupes. Les nouvelles classes et les nouveaux groupes sont ajoutés aux classes et aux groupes existants. Dans le cas d'une classe officielle: l'élève sera supprimé de sa classe officielle actuelle et à nouveau associée à la classe officielle indiquée. La méthode exige 3 paramètres:

    string $accesscode: mot de passe services web
    string $userIdentifier: champ unique utilisateur
    string $csvList: liste CSV avec un (des) code(s) classe ou un (des) code(s) groupe unique(s)

saveUserToClassesAndGroups

Cette méthode permet d'indiquer les appartenances à la classe officielle, aux classes et groupes d'un utilisateur. Il est possible d'ajouter de telles appartenances aux appartenances existantes ou de supprimer d'abord les appartenances existantes. Lors d'éventuelles erreurs pendant le traitement, un fichier CSV avec tous les codes d'erreur sera généré.

    string $accesscode: mot de passe services web
    string $userIdentifier: champ utilisateur unique
    string $csvList: liste CSV avec un (des) code(s) classe ou un (des) code(s) groupe unique(s)
    integer $keepOld: ('1' = conserver les appartenances existantes et ajouter de nouvelles appartenances, '0' = supprimer les appartenances existantes et créer uniquement les nouvelles appartenances)

sendMsg

Cette méthode permet d'envoyer un message au compte principal ou à un compte secondaire d'un utilisateur. La pièce jointe est facultatif.

    string $accesscode: mot de passe services web
    string $userIdentifier: champ unique utilisateur du destinataire
    string $title: titre du message
    string $body: texte du message
    string $senderIdentifier: champ unique utilisateur de l'expéditeur. Utilisez 'Null' afin de ne pas paramétrer d'expéditeur.
    mixed $attachments: pièces jointes (1 ou plus) dans un encodage base64 (facultatif)
    integer $coaccount: type de compte ('0' = compte principal, '1' = compte secondaire 1 ou '2' = compte secondaire 2 …)
    $coaccount est facultatif: s'il n'est pas indiqué, le message est envoyé au compte principal.
    boolean $copyToLVS: Valeur 'true' ajoute le message dans le Suivi des élèves (facultatif et valeur par défault est 'false')

Le array des pièces jointes dans le pseudo-code ressemble à ceci:

[ ['filename' ⇒ 'test1.docx', 'filedata' ⇒ 'base64encoded string'] ['filename' ⇒ 'test2.docx', 'filedata' ⇒ 'base64encoded string'] ]

Ou vous pouvez attribuer un string JSON avec la structure suivante:

[
{ “filename”: “test1.docx”, “filedata”: “base64encoded string” },
{ “filename”: “test2.docx”, “filedata”: “base64encoded string” }
]

setAccountStatus

Cette méthode permet de modifier le statut de l'élève. La méthode exige 3 paramètres:

    string $accesscode: mot de passe services web
    string $userIdentifier: champ unique utilisateur

Le dernier paramètre peut avoir les valeurs suivantes:
